# AV-R Model Credo

**Status**: FOUNDATIONAL
**Version**: 1.1.1
**Purpose**: Define the general model of how intentional change happens in Reality
**Scope**: Universal — applies to any domain where Intent carries an opinion about what Reality should be

---

## 1. The Universal Chain

> **Intent → Shaw → Reality**

Every system that processes information follows this chain. Intent determines what matters. Shaw-resolution determines
what is perceived. Reality is what exists.

AV-R is not always present. A pure calculator has Intent + Shaw but no Reality of its own. An expense tracker has
Intent + Shaw + Reality but no Vector. AV-R appears as a specific eigenstate — when Intent includes an **opinion about
what Reality should be**.

---

## 2. When AV-R Appears

AV-R manifests when Intent carries a desired state:

- **"Record what is"** → Intent + Shaw only. No opinion about R. No AV-R.
- **"Keep this as-is"** → Opinion about R (conservation). AV-R appears. Conservation intent is the hardest Agency — the
  entire security domain.
- **"Change this to that"** → Opinion about R (transformation). AV-R appears.

The trigger is not "do you want to change Reality?" but **"do you have an opinion about what Reality should be?"**

### The Gradient in Practice

Not every system handles the full chain. The same tool can shift along the gradient — a weight log without a goal
is pure R documentation; add a target and AV-R appears.

| Application                 | Shaw                 | R        | Agency role         | Vector role        | Example                                          |
|-----------------------------|----------------------|----------|---------------------|--------------------|--------------------------------------------------|
| Math software               | Transform input      | —        | —                   | —                  | Calculator, Wolfram Alpha                        |
| Plain weight log            | Show R               | Document | —                   | —                  | Scale with memory                                |
| Weight log + health tips    | Show R with opinion  | Document | Nudge toward A      | —                  | Health app with BMI warnings                     |
| Weight app + goal + booking | Show R with opinion  | Document | Nudge toward A      | Aim Force          | App that sets target, books gym, tracks workouts |
| Expense tracker (no budget) | Show R               | Document | —                   | —                  | Bank statement viewer                            |
| Expense tracker + budget    | Show R with opinion  | Document | Nudge toward A      | Aim Force          | Budget app with overspend alerts                 |
| Thermostat                  | Sense R              | Document | Built-in            | Built-in           | Fully automatic AV-R loop                        |
| Security posture tool       | Show R with policy   | Document | Conservation intent | Aim Force at drift | SIEM, compliance dashboard                       |
| Planalyzer                  | m lenses, switchable | Document | Build + track A     | Aim + align V      | The whole chain, multi-view                      |

The gradient: **Shaw-only → Shaw+R → Shaw+R+A-nudge → Shaw+R+A+V → full AV-R loop**.

A Shaw Lens that shows R *may* accidentally build Agency (you see 92kg, you care). A Shaw Lens *designed* with an
opinion nudges you toward Agency — the software becomes an **effector for Agency** (a Personal Trainer role). But
Agency requires the human to actually care. Buying a gym card gives you the possibility; it does not give you Agency.

## 3. The Two Realms

**Agency-Vector → Reality** (abbreviated **AV-R**)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  AV-OUTCOME PATH (what we DO that affects Reality)                          │
│                                                                             │
│    Agency ──────────────────────────────────────────────▶ Outcome           │
│      │ Ergon (work)                                          │              │
│      │ Repose (planned wait)                                 │ affects      │
│      │ Goal (Vector aim)                                     ▼              │
│      │ Decision (can block or split Vector)            ┌───────────┐        │
│      │                                                 │  REALITY  │        │
│      └─── Vector casts IND/MACH ──────────────────────▶│           │        │
│                                                        └───────────┘        │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│  REALITY OBSERVER PATH (unbiased observation of what IS)                    │
│                                                                             │
│    ┌───────────┐                                                            │
│    │  REALITY  │                                                            │
│    │           │──── observe ────▶ Current State                            │
│    └───────────┘                        │                                   │
│                                         │ compare                           │
│                                         ▼                                   │
│                                   Desired State                             │
│                                         │                                   │
│                                   discrepancy?                              │
│                                    ╱         ╲                              │
│                                  YES          NO                            │
│                                   │            │                            │
│                                   ▼            ▼                            │
│                              Anomaly      Goal criteria                     │
│                              detected       matched?                        │
│                                   │            │                            │
│                                   │          YES → Pragma complete          │
│                                   │                                         │
│                                   └──────▶ triggers new Agency work         │
└─────────────────────────────────────────────────────────────────────────────┘
```

Two realms, not three parts:

| Realm                  | What it is     | Contains                                             |
|------------------------|----------------|------------------------------------------------------|
| **AV** (Agency-Vector) | Capacity + Aim | Agency (what you have) + Vector (where you spend it) |
| **R** (Reality)        | What IS        | The actual state of the world, independent of AV     |

**Vector is part of AV** — Agency's spear/focus-lines aimed at Reality. Not a connector between two independent realms.
Not an edge. The arrow in "AV → R" is the model's direction, not a graph entity.

### Agency (Capacity)

Agency is the trainable capacity for intentional attention — accumulable, spendable:

- Not intent itself (Intent is upstream, pre-AV-R)
- Not work itself (work happens when Agency is spent through the Vector)
- Not attention itself (attention is what happens when you aim the Vector)

Agency is what you **have**. You build it, you accumulate it, you spend it.

### Vector (Aim)

Vector is where you **direct** your Agency at Reality. Three distinct operations:

| Operation | What it does                                       | Level                           |
|-----------|----------------------------------------------------|---------------------------------|
| **Aim**   | Point the Vector at a target (Goal)                | Single Vector, single direction |
| **Align** | Coordinate multiple Vectors toward the same target | Group, same direction           |
| **Focus** | Channel Force through the Vector                   | Intensity, concentration        |

Agency without Vector is unspent capacity (potential, not action). Vector without Agency is an empty aim (no force
behind it).

## 4. Reality and Change

### R → R'

Reality changes whether or not AV-R acts:

```
R' = R + Ri + Rd
```

| Component | What it is      | Source                                                      |
|-----------|-----------------|-------------------------------------------------------------|
| **R**     | Current Reality | What exists now                                             |
| **Ri**    | Reality-Intent  | What AV aimed to change — whether it landed or not          |
| **Rd**    | Reality-Drift   | Everything else: entropy, others' Ri, accidents, regulation |
| **R'**    | New Reality     | What Reality became                                         |

**Rd happens regardless of AV-R.** Entropy drives drift whether or not anyone acts. R → R' is physics, not the model.
AV-R adds Ri — the intentional component.

**Frame-relative**: Another team's Ri is your Rd. Observer classifies: was this change our Ri or Rd?

### Two Deltas

The system must show TWO deltas, not one:

1. **Intended change** (Ri): Did AV achieve what it aimed for?
2. **Unplanned drift** (Rd): What changed that nobody intended?

Traditional management conflates these — tracking "velocity" mixes AV progress with Reality drift.

---

## 5. Birth Mechanism

AV-R is not born linearly. It emerges through a dissonance resolution loop:

```
Intent shifts → Shaw-resolution increases (new basis)
    → Dissonance (interference between old intent I₁ and new intent I₂)
    → Dissonance pushes Intent
    → Three exits:
        Retreat    (I₂ removed, no AV-R)
        Consolidate (opinion without structure, no Pragma)
        Crystallize (I₃ = Agency, structure born, AV-R appears)
```

**Agency = Intent crystallized into structure.** "I should do something" is NOT Agency — it's Intent deepening. Agency
requires a node with edges: a plan, a structure in the model. The gap between "I should" and "here's what I will do" is
where New Year's resolutions die.

The loop can cycle multiple times before resolving. Two intents simultaneously active = dissonance. Three resolution
strategies map to Festinger (1957).

---

## 6. Anomaly

**Anomaly is general** — not specific to any implementation.

Any system where Intent carries a desired state produces Anomalies:

- Expense tracker + budget → Anomaly (over budget)
- Weight tracker + goal → Anomaly (above target)
- Thermostat + setpoint → Anomaly (temperature deviation)
- Security posture + policy → Anomaly (compliance gap)

Anomaly is the structural consequence of Intent meeting Reality. It is an **observed discrepancy** between what IS (R)
and what should be (desired state), detected independently of any AV work completing.

---

## 7. Two Paths

Two parallel paths operate independently:

### AV-Outcome Path (what we DO)

Agency builds capacity → Vector aims it at Goal → Force is spent → Reality changes (R' includes Ri).

### Reality Observer Path (what IS)

Observe R' → Compare to desired state → Discrepancy? → Anomaly detected OR Goal criteria matched.

**Key insight**: The Observer Path is **independent** of the AV-Outcome Path. We observe Reality as it IS, not just to
check our work. This prevents confirmation bias.

**Why two paths**: You can reach your Goal (patches applied) but the gap between desired state and current Reality may
have widened (new CVEs appeared). Mixing these loses information.

---

## 8. Entropy

> Reality drifts toward disorder.

Maintaining order requires:

- Continuous **observation** (sense degradation)
- Continuous **action** (restore/improve)
- Continuous **energy** (Force channeled through Vector)

Without Agency spending Force through the Vector, Reality degrades. Conservation intent (keeping R as-is) is active work
against entropy, not passive.

---

## 9. Casting

> "Effectors are cast, not born."

Casting is a general principle: selecting who/what will carry Agency through the Vector toward a Goal. Three failure
modes:

1. **No force at target** — the Effector cannot apply force where needed
2. **No Vector** — the Effector has no direction (uncast)
3. **Misaligned Vector** — the Effector's Vector doesn't align with others toward the Goal

Casting is not needed at the two model granularities (Intent → Shaw → Reality, and AV-R + R → R'). It operates at the
implementation level — how AV is staffed and aimed.

---

## 10. Perception Boundary

The AV-R model describes **perception and capacity**, not action and outcome.

Action/outcome is downstream of AV-R. Outcomes depend on Force, Authority, Structure — factors beyond the model. AV-R
predicts:

- Whether Agency has been built (capacity)
- Whether the Vector is aimed (direction)
- Whether Reality is observed independently (two paths)
- Whether Anomalies are visible (perception)

It does NOT predict whether the work will succeed. That depends on the quality of execution, external factors, and Rd.

## 11. Primitives Classification

Eight primitives across two realms:

### AV Realm — Agency (capacity)

| Primitive  | Role                                                                             |
|------------|----------------------------------------------------------------------------------|
| **Pragma** | Crystallized plan — the structured work-thing (three forms: ergon, repose, pyle) |
| **Role**   | Contextual key enabling action — you HAVE roles filled                           |

Also in Agency: Effector (available for casting — planning capacity), Patron (authority allocated), Facilitator (support
allocated), Force (energy/power available), Goal as wish (constructed aim, before Vector aimed).

### AV Realm — Vector (aim)

| Primitive        | Role                                                                 |
|------------------|----------------------------------------------------------------------|
| **Goal** (aimed) | Tip of the spear — constructed in Agency, lives on Vector once aimed |
| **Decision**     | Steering mechanism — fork, redirect, or block the Vector's path      |

Also on Vector: Focus (channeled Force — the aiming in action).

### R Realm — Reality (what IS)

| Primitive   | Role                                               |
|-------------|----------------------------------------------------|
| **IND**     | Individual human — exists, can be cast as Effector |
| **MACH**    | Machine/system — exists, can be cast as Effector   |
| **ORG**     | Organization — structural container, groups INDs   |
| **Anomaly** | Observed discrepancy — triggers new AV work        |

**Removed**: Data (general term, attachment pattern), Deliverable (dissolves into Goal + Pragma + R').

## 12. Naming Rules

| Write                             | Don't write                | Why                                                                        |
|-----------------------------------|----------------------------|----------------------------------------------------------------------------|
| **AV-R**                          | AVR, A-V-R                 | V is part of AV. Hyphen after AV makes the structural dependency explicit. |
| **Agency-Vector → Reality**       | Agency–Vector–Reality      | Written out form. Arrow = model direction, not graph edge.                 |
| **AV Realm**                      | Agency Realm               | Don't drop the Vector when referring to the AV side.                       |
| **AV Path** / **AV-Outcome Path** | Agency Path                | Same reason — V is part of AV.                                             |
| **R Realm**                       | Reality Realm (acceptable) | Short form. "Reality Realm" is acceptable but verbose.                     |

**"Agency" alone** is correct only when referring to the concept (capacity for intentional attention), NOT when
referring to the AV side of the model.
